## Feito

### Pesquisa antes de escrever código (cumprindo princípio "estuda o domínio antes de estruturar")
- Confirmado padrão atual de Supabase para Next.js App Router: `@supabase/ssr ^0.12.0` com `createBrowserClient`/`createServerClient`, função exportada via factory para evitar singleton problemático.
- Confirmado `getUser()` > `getSession()` em servidor (getUser() valida com o servidor; getSession() confia só no cookie local — risco de security bypass).
- Confirmado Next.js 16 renomeou `middleware.ts` → `proxy.ts` — descoberto no primeiro build de validação (DEC-015).
- Pesquisado e decidido: Session pooler (5432) em vez de Transaction pooler (6543) — Transaction não suporta prepared statements, causa bugs intermitentes com Drizzle (DEC-014).
- Pesquisado SSL com node-postgres + Supabase: pooler tem certificado válido, `rejectUnauthorized: true` (padrão) funciona sem ajuste extra; `rejectUnauthorized: false` desaconselhádo sem motivo (DEC-014).
- Pesquisado `@fastify/jwt` para verificação de JWT HS256 do Supabase Legacy Secret — é o caminho recomendado para "backend de terceiros", mais simples que JWKS (DEC-014).
